# 実装計画: 「すきなこと」セクションの追加 (20260310)

## 実装のゴールと要件
ユーザーから「できることセクションの下に、すきなことセクションを追加したい。良い感じのデザイン案で実装してほしい」と要望がありました。
テキストデータ（PKM, AI, 写真, ゲーム, 手帳, 植物, 猫）を使い、ポップで目を引く**「フローティング・ステッカー（ぷかぷか浮かぶシール風）」**デザインを提案・実装します。

## 変更するファイルと影響範囲
- `src/pages/index.astro`
  - 影響範囲: トップページのみ。「できること」セクションの直下に追加。

## デザイン案の詳細（フローティング・ステッカー）
- **外観**: 各「すきなこと」を角丸の長くてぷっくりしたバッジ（ステッカー）にする。
- **配色**: パステルカラー（淡いピンク、ブルー、グリーン、イエローなど）の背景色と少し濃い文字色。
- **モーション（アニメーション）**:
  - CSSで各プロパティに `@keyframes` を使い、常にゆっくりと上下にふわふわ浮遊しているように見せる。
  - ホバー時（マウスオーバー時）には少し拡大して手前にポンッと跳ねる（スケールアップ＋影が濃くなる）インタラクションを入れる。
- **レイアウト**: `flex-wrap: wrap; justify-content: center; gap: 1.5rem;` で画面中央にランダムな角度で散りばめられているような配置（少し回転 `transform: rotate` させる）。

## 具体的なステップ
1. `index.astro` 内の `features-section` の後に `<div class="likes-section">` を作成し、見出し（`section-title-tag`）と要素グループを配置。
2. 各趣味に対して専用のクラス（例: `.like-sticker`）を付与。
3. `index.astro` の `<style>` 内に新セクションとステッカー用のレイアウト、スタイル、`@keyframes`（`floating`）を追加。
4. （必要であれば）インラインスタイルまたは `nth-child` でそれぞれの色やアニメーションの遅延（`animation-delay`）をずらし、不規則に動くようにする。

## リスクと対策
- **リスク**: スマホなどの狭い画面でステッカー同士が重なりすぎて読めなくなる。
- **対策**: `gap` を適切に取り、モバイル用メディアクエリでマージンと文字サイズを最適化する。過度なアニメーションはパフォーマンスに影響するため、`transform` と `opacity` のみに制限する。
