# スマホ表示時ヘッダーレイアウトの改善

スマホ表示時のヘッダーに対して、ご要望通り「BLOG」リンクの追加とハンバーガーメニューの右側配置を行いました。

## 概要

`src/components/Header.astro` のCSSに手を加え、Flexboxの `order` プロパティを活用することで、DOMの構造を変更せずにモバイルレイアウトの視覚的な順序を制御しました。これにより、各要素を期待される位置に配置しています。

## 修正内容

*   **BLOGリンクの表示追加**:
    *   `.internal-links > a:not(...)` のCSSセレクターに `[href="/blog"]` を追加し、メニューが閉じた状態でも「BLOG」へのリンクが表示されるようにしました。
    *   追加されたリンクが画面幅に収まるよう、各リンク間の隙間 (`gap`) を `0.5rem` から `0.25rem` に少し縮めました。
*   **ハンバーガーメニューの右側配置**:
    *   `.hamburger` に `order: 2` と `margin-left: auto` を設定し、全体の右端に配置されるように調整しました。
    *   それ以外の永続リンク (`.internal-links`) には `order: 1` を設定し、左側に配置されるようにしています。
    *   ハンバーガーメニューが開いた際のリンク全体 (`.internal-links.open`) は `order: 3` となり、ヘッダーのすぐ下に全幅で展開されるようにレイアウトを整備しました。

## バリデーション

*   `npm run build` を実行し、正常に完了することを確認しました。
*   `npm run dev` が起動中のため、ブラウザから変更内容が即座にご確認いただける状態です。
